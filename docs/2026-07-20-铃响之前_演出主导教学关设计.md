# 《这里有光》序章《铃响之前》演出主导教学关设计

> 版本：v0.1
> 日期：2026-07-20
> 类型：演出主导、低门槛参与的可玩序章与基础操作教学关
> 衔接：结束后无缝进入第一章《静音的教室》

---

## 1. 设计目标

《铃响之前》让玩家在真正开始“发现同学的问题”前，先看懂这个晚自习世界是怎样运转的：谁在教室里、铃声意味着什么、老师和学生各自在承受什么、为什么一些异常很容易被日常秩序淹没。

它是一关真正的关卡，只是关卡的大部分节奏由场景演出推动。玩家始终在同一个 3D 世界里，角色、镜头、音频和 HUD 都与正式关卡一致；每当需要学习一个操作，控制权就自然回到玩家手中。完成教学关后，玩家应当能不看说明就知道：

1. 如何转动视角看向一个方向；
2. 如何短距离移动到明确目标；
3. 如何靠近并确认互动；
4. 如何暂停、打开字幕和辅助选项；
5. 本作的核心不是“找最优答案”，而是注意到、留在现场、再把事情交给可靠的人。

序章不讲解第四章的风险判断，不让玩家提前背诵心理知识，也不把任何同学的困扰当成悬念消费。它只建立环境、关系和操作信心。

**时长目标**：7 分 30 秒至 9 分钟。

**参与比例**：约 75% 场景演出／25% 轻操作。演出不是视频播放；玩家没有“看片模式”，只有临时受限或恢复的角色控制。基础操作不可失败，最长等待 12 秒后由苏念自然完成。

---

## 2. 体验原则

### 2.1 演出不是背景板

所有演出都必须推进信息，而不是用空镜头拖时长。每段演出只承担一个任务：交代时间、介绍空间、表现角色关系、解释课堂节奏，或把玩家送到下一个操作节点。

演出由游戏内 `SceneDirector` 驱动：角色路径、相机锚点、环境音和可交互热点都存在于当前场景。不得切入硬编码全屏 MP4、播放器进度条、章节选片或“按任意键跳过过场”。玩家看到的是同一个世界暂时把注意力收窄，而不是游戏暂停让他看一段片子。

### 2.2 操作一定有叙事理由

不出现“请按 W”“请按 E”这种脱离场景的考试题。每次操作都由苏念当下的需要引出：看一眼走廊、走到座位、把水杯放好、确认设置。

### 2.3 不把教学失败做成惩罚

玩家没有在提示时间内操作时，苏念会自己完成动作，场景仍保持连贯，主线继续。教学段不会被包装成可选视频；需要复习操作时，设置页提供简短的操作说明和按键映射，不提供选片式重播。

### 2.4 左侧目标只说当前要做的事

教学关沿用正式关卡的 HUD 语言。任何时刻只显示一项当前目标和一条可选提示；演出收束控制时 HUD 退为极简，但不消失为播放器界面。

## 3. 一句话故事

傍晚，苏念走进教学楼。她看见教师在走廊尽头确认晚自习安排，看见班长把教室门打开，也看见每个人都像平常一样坐下。铃响前，一切都还只是普通的一天。她回到第三排座位，放好水杯，抬头时，林澈已经坐在左边。

画面停在“晚自习开始”前半秒，第一章从这里接管。

## 4. 玩家状态与通关结果

序章不产生关系分、风险分或结局分支，只保存教学完成情况和辅助偏好。

```swift
struct PrologueState: Codable, Equatable {
    var openingViewed = false
    var lookTutorialCompleted = false
    var movementTutorialCompleted = false
    var interactionTutorialCompleted = false
    var accessibilityTutorialAcknowledged = false
    var prologueCompleted = false
}
```

### 4.1 必须写入的结果

| 结果 | 写入时机 | 后续作用 |
|---|---|---|
| `prologueCompleted` | 最后一段演出结束 | 新档从第一章开始；菜单显示操作说明 |
| 字幕、音量、减少动态效果、键盘替代输入偏好 | 玩家在序章或设置中修改时 | 全程立即生效并持久化 |
| `lookTutorialCompleted` 等 | 玩家操作或自动兜底完成时 | 不再在第一章重复弹出基础教学 |

任何未完成的教学标记都不能阻止进入第一章。它们只决定后续是否显示一次简短提示。

## 5. 完整流程

| 段落 | 时长 | 形式 | 玩家当前目标 | 玩家实际输入 | 自动推进与信息结果 |
|---|---:|---|---|---|---|
| 0. 校门外 | 0:00-0:55 | 受控入场演出 | 走进教学楼 | 只允许暂停与设置 | 交代傍晚、教学楼、晚自习将开始；苏念自动进入楼内 |
| 1. 看一眼走廊 | 0:55-1:35 | 演出交还视角 | 看向走廊尽头 | 转动视角 | 认识“视角决定你注意到什么”；12 秒后苏念自然转头 |
| 2. 走回座位 | 1:35-2:20 | 轻移动 | 回到第三排座位 | WASD 或点击路径 | 认识短距离移动与自动路径辅助；12 秒后苏念自动走回 |
| 3. 放好水杯 | 2:20-2:55 | 轻交互 | 把水杯放到桌上 | 靠近并确认 | 认识热点和确认键；12 秒后苏念自行放下 |
| 4. 晚自习如何开始 | 2:55-5:10 | 教室内受控演出 | 留在座位，听教室安静下来 | 可转头、暂停、开设置 | 展示老师、班长、同学与教室节奏；玩家可看但不需要选择 |
| 5. 看见左边的人 | 5:10-5:50 | 受控视角 | 看看左边的林澈 | 转向左侧并停留 | 让玩家第一次练习“注意到”而非“判断”；10 秒后自动完成 |
| 6. 先把自己准备好 | 5:50-6:25 | 轻交互 | 试着深呼吸一次 | 确认呼吸或跳过 | 介绍自我照顾不是额外任务；10 秒后以安静坐好完成 |
| 7. 暂停与辅助 | 6:25-7:05 | 受控停顿 | 确认你可以随时调整体验 | 打开设置或确认继续 | 展示字幕、音量、减少动态效果、键盘替代输入；未打开也可确认 |
| 8. 铃响前半秒 | 7:05-8:00 | 场景演出转正式关卡 | 等待晚自习开始 | 可转头、暂停 | 林澈坐下、教室灯光稳定、预备铃响；HUD 直接切为第一章 |

所有段落的最长总时长不超过 8 分钟。演出中的可看区域可自由观察，但玩家不能离开当前教学范围；这样它仍是关卡，而不是被动观看。

## 6. 分镜与教学细节

### 6.1 段落 0：校门外

**画面**：固定广角，从校门外看向亮灯的教学楼。学生陆续进入，不给任何人特写。镜头跟随苏念穿过走廊窗边，落在晚自习时间表上。

**声音**：远处球场收场声、楼道脚步、广播试音。字幕只写方向和环境声，例如“[楼道远处：脚步声]”，不在开场就提示谁情绪异常。

**苏念旁白**：

> “晚自习开始前，楼里总有一小段时间。大家在找座位，也在把白天没收好的东西先藏起来。”

**教学意图**：让玩家先理解“普通”是什么样子，后续才有能力看见不普通的地方。

### 6.2 段落 1：看一眼走廊

**切换**：镜头从演出镜头落到苏念第一人称。HUD 左侧淡入：

```text
主线任务
熟悉晚自习开始前的教室

当前目标
看向走廊尽头

提示
移动鼠标或使用视角键
```

走廊尽头，方老师在和班长低声确认值日安排。玩家只需把镜头转到目标区域，画面给一个很轻的聚焦反馈，便立即完成。

**自动兜底**：12 秒未完成时，苏念自然侧身让路并向走廊看一眼。HUD 写“你注意到了走廊里的动静”，随后进入下一段。不得弹出“操作失败”。

### 6.3 段落 2：走回座位

**目标**：第三排座位出现低调的桌灯轮廓，地面只出现一条短路径，不显示地图、箭头雨或发光收集物。

**操作**：玩家可用 WASD 或点击座位前的路径点。角色移动距离不超过 6 米，途中没有碰撞谜题。靠近座位后自动收起移动控制，镜头转为固定座位第一人称。

**自动兜底**：12 秒未移动，苏念说“先回座位吧”，角色自动走过去。以后正式关卡不会自动代替普通移动；这里只是保证教学不断。

### 6.4 段落 3：放好水杯

**目标**：桌面上有水杯和摊开的笔记本。水杯轻微高亮，确认键图标只显示一次。

**操作**：看向水杯，按确认键。镜头不需要精准瞄准到像素级，热点有宽容范围。

**苏念旁白**：

> “水放在手边。等会儿如果脑子很乱，至少不用再找它。”

这是对第一章“自我照顾”的无压力预告，不计分、不恢复数值，也不强制玩家理解其意义。

### 6.5 段落 4：晚自习如何开始

这是教学关最长的一段**教室内实时演出**，约 2 分 15 秒。玩家仍坐在第三排，可以缓慢转头看向左、右、前方；不能起身、不能触发互动。镜头不从“全知上帝”角度揭露任何人的隐私，只呈现公共空间中能被看见的行为。

| 镜头 | 时长 | 画面信息 | 声音信息 |
|---|---:|---|---|
| 教师 | 25 秒 | 方老师把手机扣在讲台边，检查最后一排空位 | 讲台附近的低声安排、翻页声 |
| 班干部 | 25 秒 | 周予安关门、核对座位；许栀把掉在地上的笔递回去 | 门轴声、简短“给你” |
| 同学 | 35 秒 | 有人写题、有人趴一下后坐直、有人把耳机收进抽屉 | 笔尖、椅子、压低的咳嗽 |
| 苏念 | 20 秒 | 苏念从左到右扫过教室，镜头不在任何人身上停得过久 | 她的呼吸、灯管嗡声 |
| 林澈 | 30 秒 | 林澈在左侧坐下，把书翻到一页；不做异常特写 | 纸张摩擦，环境声重新占满 |

**结论不是明说的台词，而是演出感受**：教室很安静，但安静不等于每个人都没事。

### 6.6 段落 5：看见左边的人

HUD：

```text
当前目标
看看左边的林澈

提示
先看见，不急着判断
```

玩家向左看并停留约 1 秒即可完成。林澈只是在翻书，没有哭、没有触发事件，也没有“正确答案”。苏念旁白：

> “有时候你先看见的，只是一个人比平常安静。”

这段建立本作的观察语言：玩家学习的是停下来注意，不是用一个镜头就给人下结论。

### 6.7 段落 6：先把自己准备好

HUD：

```text
当前目标
试着深呼吸一次

提示
不想操作也没关系，先坐一会儿
```

玩家可按呼吸键或选择“先坐一会儿”。两者都完成教学，不记录为第一章的 `suNianSelfCared`，避免序章替玩家完成正式关卡的选择。

### 6.8 段落 7：暂停与辅助

教室演出自然停在预备铃前，右上角暂停图标轻微亮起；非遮挡式设置提示从右下角出现。不是“暂停影片”，而是告诉玩家正式游玩中同样随时可以调整体验：

```text
你可以随时暂停
调整字幕、声音、动态效果和输入方式
```

玩家可以打开设置，也可以按“继续”。若打开设置，首次展示四项已能生效的控件：

| 控件 | 默认 | 教学说明 |
|---|---|---|
| 方向字幕 | 开启 | 声音线索以方向和强度文字呈现 |
| 音量 | 70% | 环境、对白和提示音分别调整 |
| 减少动态效果 | 跟随系统 | 镜头推拉和转场改为淡入 |
| 键盘替代输入 | 开启 | 不依赖鼠标拖动完成微游戏 |

### 6.9 段落 8：铃响前半秒

**画面**：设置提示收起。苏念看向前方；方老师没有讲话，只把灯调亮一点；林澈翻开书；预备铃在远处响起。

**旁白**：

> “铃响以后，大家都会开始做自己的事。也许我也是。”

镜头保持在第三排座位第一人称。第一章 HUD 不淡出黑屏，而是在铃声余韵中替换：

```text
主线任务
发现今晚不对劲的地方

当前目标
看看林澈今晚在做什么
```

## 7. 系统与流程规则

### 7.1 演出关卡状态机

每段演出是具名 Beat，不使用“播放完回调里直接推进下一段”的隐式链条。

```swift
enum PrologueBeatID: String, Codable, CaseIterable {
    case gateArrival, lookDownHall, returnToSeat, placeWater
    case studyHallRhythm, noticeLinChe, settleBreath
    case accessibility, bellBeforeClass
}
```

`GameManager` 持有唯一的 `PrologueState` 和当前 Beat。`SceneDirector` 驱动 NPC 路径、相机锚点、音频和输入限制；它不能自行写剧情状态。玩家输入、自动兜底、演出完成都调用同一条幂等 `completePrologueBeat` 路径。

### 7.2 暂停、读档与自动推进

- 暂停时受控演出、字幕计时、输入窗口和自动兜底一起冻结；环境循环可以继续。
- 每一段完成后创建稳定检查点。读档只回到段落开头或已完成后的下一个段落，不恢复到半秒钟的镜头转场中间。
- 已看过序章的玩家可以在第二次开新档时选择“进入序章”或“直接从第一章开始”；直接开始时默认所有基础教学已知，但辅助设置仍可在任何时刻打开。
- 不提供第一次游玩的“一键跳过全部操作”。玩家至少会看到并确认一次视角、移动、交互和辅助设置提示；每一步可以自动完成。

### 7.3 输入限制

| 阶段 | 允许输入 | 禁止输入 | 原因 |
|---|---|---|---|
| 受控入场演出 | 暂停、字幕、音量、返回菜单 | 移动、剧情选择 | 保持镜头语言和节奏 |
| 视角教学 | 转头、暂停 | 自由移动、菜单以外互动 | 让注意力放在视角上 |
| 移动教学 | 移动、暂停 | 跑步、离开短路径、任意热点 | 不把教学变成跑图 |
| 交互教学 | 看向热点、确认、暂停 | 其他物品交互 | 让确认键的结果明确 |
| 结尾转场 | 暂停、设置 | 移动、互动 | 避免第一章前出现控制断裂 |

## 8. 美术、声音与字幕

### 8.1 视觉语言

- 夕阳到灯光：序章从偏暖的室外过渡到中性教室灯，不使用恐怖化滤镜。
- 镜头保持贴近苏念，不频繁切特写。人物被拍到时以公共行为为主，避免替玩家窥探隐私。
- HUD 面板沿用正式关卡的 Liquid Glass 语言，圆角不超过 8pt；受控演出段只保留小型暂停和字幕按钮，绝不出现播放器控件。
- 减少动态效果开启时，推轨、抖动和快速转场替换为 0.2-0.4 秒淡入淡出。

### 8.2 声音语言

本关用声音建立“正常晚自习”的基线：灯管、风扇、笔尖、椅子和远处广播。对白不超过必要程度，约 70% 的环境信息由声音承载，但每个必要声音都配有可关闭方向字幕。

游戏设置支持独立调节：对白、环境、提示音；字幕不自动暴露未在画面或任务中公开的人物身份。

## 10. 验收标准

- [ ] 首次进入后 3 秒内出现画面或字幕，不出现黑屏等待。
- [ ] 玩家在不操作时，最多 12 秒也能完成任一教学节点并进入下一段。
- [ ] 视角、移动、交互、暂停／辅助四类基础操作都各有一次自然情境下的提示。
- [ ] 任一教学操作都没有失败、扣分、重来或“错误答案”。
- [ ] 演出暂停后，字幕、音频、自动兜底和镜头进度一致冻结；恢复后不跳段。
- [ ] 开启方向字幕、减少动态效果、键盘替代输入后，序章可完整结束。
- [ ] 序章结束时，玩家处于第一章的第三排座位初始镜头、初始 HUD 和初始环境音，不出现加载黑屏。
- [ ] 已完成序章的新档直入第一章不会改写六章的关系、风险、同伴或结局状态。
- [ ] 演出中不把同学的隐私、心理状态或风险当作可供围观的谜题展示。

## 11. 给 AI 实现的边界

1. 先实现 Beat 状态机、稳定检查点、暂停和自动兜底，再接镜头和角色演出。
2. 不要把序章做成独立播放 App 或影片选择器；它必须从同一个 `GameManager`、设置和存档系统进入正式第一章。
3. 不要在演出中重复构建一套 HUD、设置或输入系统；教学只调用正式系统的受限子集。
4. 不要用“按键说明卡”替代场景中的操作动机，也不要为增加可玩性塞入收集物、跑图或对错选择。
5. 不使用预渲染过场替代关卡演出。运行时状态、镜头、NPC、热点和自动推进都由同一套游戏状态机驱动。

---

*本序章的目的不是在故事开始前解释一切，而是让玩家先安静地进入这个教室，并在真正需要注意时，知道自己该怎样看、怎样走、怎样停下来。*
